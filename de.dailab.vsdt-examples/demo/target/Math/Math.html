Process Diagrams to be transformed to both BPEL and JIAC

Extract Root

This diagram contains a few pools representing simple algorithms. While algorithms surely are not a typical usecase for BPMN, they do well for presenting the capabilities of the VSDT. The algorithms can equally well be translated to BPEL and JIAC, they can be interpreted by the VSDT's built-in interpreter and a textual representation can be generated.

Mathematican

The Pool 'Mathematican' belongs to Participant 'Mathematican'. In this process the square root of the input value is approximated. The Pool's Process is of type private. In that Process, a number of elements are executed in sequence.
First, in Event 'start' the Message 'GetSquareRootRequest' of the Service 'TestService.SquareRoot' is received. The input message is received and its payload is bound to the respective process properties. After that, starting at the Gateway 'input valid?', one out of several branches will be accomplished:
If the condition '(number < 0) and (eps <= 0)' holds true, in Event 'invalid input' the Message 'GetSquareRootResponse' of the Service 'TestService.SquareRoot' is sent. In case of invalid input, the return value will be zero. Further, an error message is returned. If no condition applies, by default a number of elements are performed in sequence.
First, the Task 'initialize variables' is performed. This initializes the variables: The upper and lower bounds and the currently tested number. Afterwards, starting at the Gateway '_2', a part of the Process will be performed in a loop. At first, a number of elements are processed in sequence.
First, the script Task 'print' is processed, running a script. Here, the currently tested number is printed. The script's body can not be translated to BPEL, thus this works only when translating to JIAC. After that, the Task 'calculate delta' is performed. Delta is the difference between the input number and the square of the currently tested number.
After that, the loop condition will be tested. While the condition ((d < 0) and ((0 - d) > eps)) or ((d > 0) and (d > eps)) holds true, the loop will continue and starting at the Gateway 'too large?', one out of several branches will be performed:
If the condition '(current * current) > number' holds true, the Task 'try smaller number' is conducted. The currently tested number is used as the new upper bound and is then set to be the average of itself and the lower bound. If no condition applies, by default the Task 'try greater number' is performed. The currently tested number is used as the new lower bound and is then set to be the average of itself and the upper bound.
Afterwards, in Event 'return result' the Message 'GetSquareRootResponse' of the Service 'TestService.SquareRoot' is sent. Returns the result of the approximation and a success message.

Mathematican

The Pool 'Mathematican' belongs to Participant 'Mathematican'. The Pool's Process is of type private. In that Process, a number of elements are conducted in sequence.
First, in Event 'start' the Message 'GetFacultyRequest' of the Service 'TestService.Faculty' is received. Afterwards, starting at the Gateway 'check input', one out of several branches will be performed:
If the condition 'n >= 0' holds true, a number of elements are carried out in sequence.
First, a part of the Process will be accomplished in a loop. At first, the Task '_DO_NOTHING_' is executed. Hereupon, the loop condition will be tested. While the condition n > 1 holds true, the loop will continue and the Task 'Multiply' is performed. After that, the Message 'GetFacultyResponse' of the Service 'TestService.Faculty' is sent.
If no condition applies, by default the Message 'GetFacultyResponse' of the Service 'TestService.Faculty' is sent.
