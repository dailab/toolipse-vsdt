This Diagram is used for testing the Text Generation Capabilities of the VSDT. Here the focus is not so much on executability but on good documentation.

Answering Support Call

This Business Process is about handling incoming support calls.

Customer Support

This Pool illustrates the process of Customer Support. The Pool 'Customer Support' belongs to Participant 'Customer Support'. This Pool is made up of 2 Lanes:
  • Help Desk: This Lane represents the Help Desk, which receives all incoming trouble tickets and will look up the answers for known problems or escalate the ticket to the technical support.
  • Technical Support: The Technical Support receives tickets that can not be handled by the help desk, e.g. new problems.
The Pool's Process is of type private. In that Process, a number of elements are processed in sequence.
First, in Event 'Receive Ticket' the Message 'HelpDeskQuestion' of the Service 'CS.askHelpdesk' is received. Afterwards, starting at the Gateway 'Known Problem?', one out of several branches will be accomplished:
If the condition ''unknown problem'' holds true, a number of elements are performed in sequence.
First, an interruptable region begins.
Here, the manual Task 'Try to reproduce Problem' is conducted. This section can be interrupted if in Event 'Timeout' the Activity is interrupted after '30 minutes'.
Hereafter, a number of elements are accomplished in sequence.
First, starting at the Gateway 'Kind of Problem?', one or more out of several branches will be carried out:
If the condition ''is bug'' holds true, the send Task 'Inform Developer' is processed, sending the Message 'BugTrackerTicket' which belongs to the Service 'Dev.fileBug'. If the condition ''is stupid user'' holds true, the user Task 'Describe Solution' is performed, invoking a Service.
After that, the send Task 'Add to Database' is conducted, sending a Message.
This part will only be carried out if the timer Event 'Timeout' has not been triggered
If no condition applies, by default the user Task 'Look up answer in Database' is performed, invoking the Service 'Database.getEntry' with the input Message getEntryRequest and the output Message getEntryResponse.
Hereupon, the send Task 'Send Answer' is carried out, sending the Message 'HelpDeskResponse' which belongs to the Service 'CS.askHelpdesk'. Hereafter, the user Task 'File Ticket' is conducted, invoking a Service. Next, in Event 'End Process' nothing special happens.

Development

This Pool stands for the Development department, being further distinguished into bugfixing and others. The Pool 'Development' belongs to Participant 'Development'. The Pool's Process is of type collaboration. In that Process, a number of elements are conducted in sequence.
First, in Event 'Receive Bug' the Message 'BugTrackerTicket' of the Service 'Dev.fileBug' is received. Afterwards, an interruptable region begins.
Here, the ad-hoc embedded Subprocess 'Work day and night to fix the bug' is performed. In this course, the manual Task 'Code' is performed. Furthermore, the manual Task 'Get Coffee' is accomplished. Aside from that, the normal flow and the exceptional flow are merged in the Gateway 'INITIAL_GATEWAY'., which references to another Process in another Business Process Diagram. This section can be interrupted if in Event 'Go insane' an error with error code 'nuts' is caught. The poor developer went insane. In this case, the send Task 'Inform HR' is processed, sending a Message.
Hereupon, the user Task 'Mark bug as being resolved' is conducted, invoking a Service. This part will only be performed if the error Event 'Go insane' has not been triggered After that, the normal flow and the exceptional flow are merged. Following that, nothing special happens.

Customer

This is a blackbox Pool which represents the customer making the call at the help desk. No internal process is shown for this Pool. The Pool 'Customer' belongs to Participant 'Customer'. The Pool's Process is of type abstract.
